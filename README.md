# python_basic_grammar

ProgateでPython基礎文法編を学んだ内容を応用して、日常の意思決定や業務ロジックを模したプログラムを2つ作成しました。

- `convenience_store.py`：コンビニ会計シミュレーター（単品）
- `eating_out.py`：外食していい？判定プログラム（厳しめ）

---

## 1. コンビニ会計シミュレーター（単品）｜`convenience_store.py`

購入したい商品を1つ選び、所持金・購入個数・割引・クーポン適用の有無に応じて、支払い可否やお釣りを表示します。  
店員モードを選ぶと、売上と利益も表示します。

### 機能（できること）
- 商品を1つ選択（`0:おにぎり / 1:お茶 / 2:りんご`）
- 所持金から「最大購入可能個数」を計算して表示（整数除算 `//`）
- 購入個数に応じた割引
  - 4個以下：割引なし
  - 5〜9個：10%割引
  - 10個以上：20%割引
- 合計金額が500円以上の場合、クーポン（100円引き）を適用可能
- 支払い結果の表示（購入できる／財布が空／お金が足りない）
- 店員モードの場合：売上と利益を表示（利益 = 売上 - 原価×個数）

### 入力と出力の流れ
1. 商品を選ぶ（0/1/2）
2. 所持金を入力
3. 購入する個数を入力
4. （合計が500円以上の場合）クーポン使用の有無を入力
5. 支払い結果を表示
6. 店員かどうかを入力し、店員なら売上・利益を表示

### 工夫した点
- 金額が小数（`.0`）にならないよう、割引計算を `* 9 // 10` / `* 8 // 10` で整数のまま処理した
- `input()` が文字列を返す点を意識し、`int()` で型変換してから比較・計算するようにした
- 不正な商品番号（0/1/2以外）の場合は処理を進めずメッセージを表示するようにした

## 2. 外食していい？判定プログラム（厳しめ）｜`eating_out.py`

月間の食費予算と現在の支出、残り日数から「1日平均の食費上限」を算出し、  
外食は嗜好性が高い前提で **60%** に制限して、今日の外食が許容範囲かを判定します。

### 仕様（厳しめルール）
- 外食上限 = 1日平均の食費上限 × 60%（小数を避けるため `* 6 // 10` で計算）
- 残り予算がマイナス（赤字）の場合は即NG
- 外食上限が300円未満の場合もNG
- 閏年は考慮していない（2月は28日固定）

### 入力と出力の流れ
1. 今月の食費予算（円）
2. 今月ここまでの食費実績（円）
3. 現在の月と日付（残り日数を計算）
4. 今日の外食予定額（円）
5. 残り予算・1日平均上限・外食上限を表示し、判定（OK/NG）と理由を出力

### 工夫した点
- 円計算が小数にならないよう、比率計算を `* 6 // 10` で整数のまま処理した
- `days_left <= 0` のときに先に判定し、0で割ってエラーにならないようにした
- NGのときは「赤字」「外食上限が低すぎる」など理由も出すようにした

## 学んだこと（共通）
- 変数の定義、四則演算
- `input()` による入力の受け取りと `int()` による型変換
- `if / elif / else` による条件分岐（複数条件、入れ子）
- `//`（整数除算）による切り捨て
- 値の表示（`str()` で文字列に変換して出力）

---

## 今後の改善点（やりたいこと）
- 入力が不正な場合（例：文字入力、範囲外入力）に、もう一度入力を促す（繰り返し処理で実装したい）
- 購入可能個数（`count_max`）を超える個数を入力した場合に警告を出す
- クーポン入力を `yes/no` など文字列でも受け付けるようにする
- 商品マスタ（商品名・価格・原価）をまとめて管理し、追加しやすくする（クラスや辞書の活用）
